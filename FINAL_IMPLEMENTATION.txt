How It Works:
-------------
When you run: cicd init --platforms github --cloud-provider <provider> --deployment-type <type>

The GitHub Actions workflow automatically includes:

LOCAL (default):
- Builds Docker image locally
- Saves as artifact for download
- No cloud deployment steps

AWS:
- Configure AWS credentials
- Login to Amazon ECR
- Build and push Docker images to ECR
- Deploy to AWS App Runner (webapp) OR EC2 (instance)

AZURE:
- Login to Azure Container Registry (ACR)
- Build and push Docker images to ACR
- Deploy to Azure App Service (webapp) OR Azure VM (instance)

GCP:
- Authenticate to Google Cloud
- Configure Docker for GCR
- Build and push Docker images to GCR
- Deploy to Cloud Run (webapp) OR Compute Engine (instance)
